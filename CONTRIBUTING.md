# Contributing to MemoryBridge

Thank you for your interest in MemoryBridge! We welcome contributions from the community.

## Code of Conduct

By participating in this project, you agree to abide by standard open-source codes of conduct. Be respectful, inclusive, and constructive.

## How to Contribute

1.  **Fork the Repository:** Create your own fork of the `memorybridge` repository.
2.  **Create a Branch:** Create a feature branch for your changes (`git checkout -b feature/amazing-feature`).
3.  **Make Changes:** Implement your feature, bug fix, or documentation improvement.
4.  **Test:** Ensure the existing tests pass and add new tests for your feature if applicable.
    *   To run backend tests: `cd services/mcp-routines && pytest`
    *   To run web tests: `cd apps/web && npm run test` (if applicable)
5.  **Commit:** Commit your changes with clear, descriptive commit messages.
6.  **Push:** Push your branch to your fork.
7.  **Submit a Pull Request:** Open a pull request against the `main` branch of the original repository.

## Development Environment Setup

Please refer to the `README.md` for detailed instructions on setting up the local development environment, including Next.js, FastAPI, Google ADK, and Neon PostgreSQL.

## Bug Reports and Feature Requests

If you find a bug or have a feature idea, please open an issue on GitHub.
Provide as much detail as possible, including steps to reproduce bugs and clear rationale for feature requests.

**Important:** Please review `SECURITY.md` regarding reporting security vulnerabilities. Do not post sensitive information in public issues.
